## 当前系统架构概述

| 组件 | 当前状态 |
|------|---------|
| **数据源** | yfinance (主), Tiger API, DefeatBeta, Alpha Vantage, Tushare |
| **支持市场** | 美股 (主), 港股 (部分), A股 (仅股票) |
| **期权分析** | 4种策略评分 + VRP计算 + 风险调整 + Greeks分析 |

### 核心分析模块

- **策略评分**: Sell Put, Sell Call, Buy Call, Buy Put
- **高级分析**: VRP (波动率风险溢价), 风险调整器
- **趋势分析**: ATR计算, 支撑/阻力位
- **风险画像**: 收益风险特征分类

### 数据适配器架构

```
MarketDataService (单例)
    ├── YFinanceAdapter (优先级10, US/HK)
    ├── TigerAdapter (优先级15, US/HK/CN)
    ├── DefeatBetaAdapter (优先级20, 本地DuckDB)
    ├── TushareAdapter (优先级10-CN, 仅股票)
    └── AlphaVantageAdapter (优先级30, 备用)
```

## 1. A股场内期权 (ETF/股指)

### 产品定位

最为稳健的收租战场，流动性极佳，适合大资金作为底仓。

### 推荐标的

| 标的 | 代码 | 用途 |
|------|------|------|
| 50ETF | 510050 | 核心底仓 |
| 300ETF | 510300 | 核心底仓 |
| 中证1000 | MO | 避险用 |

### 核心策略

- 卖出宽跨式 (Short Strangle)
- 备兑策略 (Covered Call)

### 优势与风险

| 优势 | 风险 |
|------|------|
| 世界级流动性，几乎无滑点 | Pin Risk (行权指派) 是最大隐患 |
| 规则极其透明 | |
| 股指期权现金交割无接货风险 | |

### 数据可用性评估: 需要新数据源

| 数据源 | 支持状态 | 说明 |
|--------|---------|------|
| **Tushare** | 不支持 | `Tushare doesn't provide options data for A-shares` |
| **yfinance** | 不支持 | 不覆盖中国交易所期权 |
| **Tiger API** | 部分 | 支持CN市场，但期权数据需验证 |
| **天勤TqSdk** | 可用 | 免费提供50ETF/300ETF期权实时行情 |
| **金数源** | 可用 | 提供历史tick/分钟/日线数据 |
| **上交所官方** | 可用 | 提供技术接口文档 |

**推荐方案**: 集成 **TqSdk** 作为A股期权数据源

### 算法兼容性: 高度兼容

| 算法模块 | 兼容性 | 需要调整 |
|----------|--------|---------|
| 策略评分 | 兼容 | 无 - 通用Black-Scholes模型 |
| Greeks计算 | 兼容 | 无 - 交易所直接提供 |
| VRP分析 | 兼容 | 需要历史IV数据 |
| 趋势分析 | 需调整 | 需调整阈值 (A股波动率不同) |

关键差异: 股指期权为现金交割、T+0交易规则、涨跌停限制不同、无风险利率使用中国国债收益率。

### 容量评估: 极佳

| 标的 | 日均成交量 | 流动性评级 |
|------|-----------|-----------|
| 50ETF期权 | ~200万张 | 世界前三 |
| 300ETF期权 | ~150万张 | 世界级 |
| 中证1000期权 | ~50万张 | 优秀 |

## 2. 港股期权

### 产品定位

高波动率带来的高权利金，但必须严格筛选头部资产。

### 推荐标的

| 标的 | 代码 | 特点 |
|------|------|------|
| 腾讯 | 0700.HK | 周权可用 |
| 阿里 | 9988.HK | 周权可用 |
| 美团 | 3690.HK | 周权可用 |

### 核心策略

- 白名单制度
- 卖出虚值看跌 (OTM Sell Put)

### 数据可用性: 现有基础设施可支持

| 数据源 | 支持状态 | 说明 |
|--------|---------|------|
| **yfinance** | 部分 | 支持.HK股票，期权链API可能受限 |
| **Tiger API** | 支持 | 已配置 `supported_markets=[Market.US, Market.HK, Market.CN]` |
| **HKEX官方** | 支持 | 提供期权详情、CSV导出 |

**推荐方案**: 优先使用 **Tiger API**，yfinance作为备选

### 算法兼容性: 完全兼容

现有市场参数已配置:
```python
'HK': {
    'min_daily_volume': 2000000,
    'risk_premium': 1.15,
    'pe_threshold_high': 35,
    'pe_threshold_very_high': 50,
    'volatility_adjustment': 1.1
}
```

### 容量评估: 良好（头部资产）

必须实施**白名单制度**，仅允许高流动性标的入池。

## 3. 商品期权

### 产品定位

仅限特定品种的防御性准入，必须极其小心交割月规则。

### 推荐标的

| 品种 | 代码 | 交易所 | 风险等级 |
|------|------|--------|---------|
| 豆粕 | M | DCE | 中 |
| 白糖 | SR | CZCE | 中 |
| 铁矿石 | I | DCE | 高 (慎入) |

### 数据可用性: 需要全新基础设施

| 数据源 | 支持状态 | 说明 |
|--------|---------|------|
| **CTP接口** | 可用 | 需要期货公司账户 |
| **天勤TqSdk** | 可用 | 免费提供商品期权行情 |
| **真格量化** | 可用 | 期货期权实盘平台 |

### 算法兼容性: 需要重大修改

- 需要商品特定因子 (季节性、基差)
- 需要全新的商品趋势模型
- **临近交割月强平风险** 是最大死穴，需要新监控模块

## 具体实施方案

### Phase 1: 港股期权 (快速启动)

目标 2-3 天内上线：
1. 验证Tiger API的HK期权数据完整性
2. 添加港股期权白名单
3. 调整前端支持.HK后缀

### Phase 2: A股ETF期权 (新适配器)

目标 1-2 周内上线：
1. 集成TqSdk库
2. 创建TqSdkAdapter适配器
3. 添加A股期权市场参数
4. 调整前端支持新标的

### Phase 3: 商品期权 (深度开发)

目标 3-4 周内上线：
1. 扩展TqSdk适配器支持商品期权
2. 创建交割月风险监控模块
3. 开发商品特定评分算法
4. 实现主力合约自动识别
5. 添加季节性分析因子
